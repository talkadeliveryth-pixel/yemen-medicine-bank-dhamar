# Contributing Guidelines - دليل المساهمة

## شكراً لاهتمامك بالمساهمة! 🙏

نرحب بجميع المساهمات والاقتراحات لتحسين هذا المشروع.

---

## 📋 كيفية المساهمة

### 1. الإبلاغ عن الأخطاء (Bug Reports)

إذا وجدت خطأ:

1. تحقق من أنه لم يتم الإبلاغ عنه من قبل
2. افتح [Issue جديد](https://github.com/talkadeliveryth-pixel/yemen-medicine-bank-dhamar/issues)
3. اشرح المشكلة بوضوح:
   - ما الذي حدث؟
   - ما المتوقع أن يحدث؟
   - خطوات إعادة الإنتاج
   - لقطات شاشة إن أمكن

### 2. اقتراح ميزات جديدة (Feature Requests)

لديك فكرة رائعة؟

1. افتح [Discussion جديدة](https://github.com/talkadeliveryth-pixel/yemen-medicine-bank-dhamar/discussions)
2. اشرح الميزة المقترحة
3. اشرح الفائدة منها

### 3. تقديم تحسينات (Pull Requests)

تريد المساهمة بكود؟

#### الخطوات:

1. **Fork المستودع**
   ```bash
   # اضغط على Fork في أعلى الصفحة
   ```

2. **استنساخ نسختك**
   ```bash
   git clone https://github.com/YOUR_USERNAME/yemen-medicine-bank-dhamar.git
   cd yemen-medicine-bank-dhamar
   ```

3. **إنشء فرع جديد**
   ```bash
   git checkout -b feature/اسم-الميزة
   # أو
   git checkout -b fix/اسم-الإصلاح
   ```

4. **قم بالتغييرات**
   - اتبع أسلوب الكود الموجود
   - اكتب تعليقات واضحة
   - اختبر التغييرات

5. **Commit التغييرات**
   ```bash
   git add .
   git commit -m "وصف مختصر للتغييرات"
   ```

6. **Push للفرع**
   ```bash
   git push origin feature/اسم-الميزة
   ```

7. **فتح Pull Request**
   - اذهب إلى المستودع الأصلي
   - اضغط "New Pull Request"
   - اشرح التغييرات

## 📝 معايير الكود

### JavaScript
- استخدم `const` و `let` بدلاً من `var`
- اكتب تعليقات للدوال المعقدة
- استخدم أسماء متغيرات واضحة

### HTML
- استخدم الـ RTL للعربية
- اكتب `lang="ar"`
- استخدم semantic HTML

### CSS
- استخدم متغيرات CSS
- رتب الخصائص بشكل منطقي
- أضف تعليقات للأقسام

قبل تقديم Pull Request:

```bash

npm install

# تشغيل التطبيق
npm start

# اختبر التطبيق يدويًا
```
