# تطبيق بنك الدواء اليمني - فرع ذمار

## 📋 نبذة عن المشروع

تطبيق ويب شامل لإدارة بيانات المرضى والأدوية في بنك الدواء اليمني - فرع ذمار. يتميز بواجهة عربية حديثة وسهلة الاستخدام، مع قاعدة بيانات محلية للعمل بدون إنترنت.

## ✨ الميزات الرئيسية

- ✅ **إدخال بيانات المرضى** بشكل منظم وسهل
- ✅ **إدارة بيانات الأدوية** المصروفة لكل مريض
- ✅ **نموذج متكامل** يتضمن بيانات المستفيد والأدوية والوثائق
- ✅ **البحث والتصفية** عن المرضى بسهولة
- ✅ **التقارير والإحصائيات** لمتابعة العمل
- ✅ **استيراد/تصدير Excel** لتبادل البيانات
- ✅ **لوحة تحكم** توضح الإحصائيات الأساسية
- ✅ **طباعة** سجلات المرضى
- ✅ **واجهة RTL عربية** 100%
- ✅ **عمل Offline** (بدون إنترنت)

## 🛠️ التقنيات المستخدمة

### Backend
- **Node.js + Express** - خادم التطبيق
- **SQLite** - قاعدة البيانات المحلية

### Frontend
- **HTML5** - البنية الأساسية
- **CSS3** - التصميم والتنسيق
- **JavaScript Vanilla** - الوظائف والتفاعلية

### Libraries
- **XLSX** - للتعامل مع ملفات Excel
- **Multer** - لتحميل الملفات
- **CORS** - للتعامل مع طلبات Cross-Origin

## 📦 المتطلبات

- Node.js (الإصدار 12 أو أحدث)
- npm (مدير الحزم)

## 🚀 التثبيت والتشغيل

### 1. استنساخ المستودع
```bash
git clone https://github.com/talkadeliveryth-pixel/yemen-medicine-bank-dhamar.git
cd yemen-medicine-bank-dhamar
```

### 2. تثبيت المكتبات
```bash
npm install
```

### 3. تشغيل التطبيق
```bash
npm start
```

التطبيق سيعمل على: `http://localhost:3000`

## 📱 الاستخدام

### الشاشات الرئيسية

#### 1. لوحة التحكم (Dashboard)
- عرض إجمالي المرضى
- عرض إجمالي الأدوية المصروفة
- معلومات سريعة عن النشاط

- عرض جميع المرضى المسجلين
- خيارات للعرض والتعديل والحذف

#### 4. البحث
- البحث عن المريض باسمه أو رقم بطاقته

#### 5. التقارير
- إحصائيات عامة عن النشاط

#### 6. استيراد/تصدير
- تصدير البيانات إلى Excel
- استيراد البيانات من ملف Excel

## 📊 هيكل قاعدة البيانات

### جداول البيانات

#### 1. Patients (المريضى)
```
- id (معرف فريد)
- patient_number (رقم المريض)
- patient_name (اسم المريض)
- gender (الجنس)
- age (العمر)
- address (العنوان)
- phone (رقم التواصل)
- card_type (نوع البطاقة)
- card_number (رقم البطاقة)
- disease_description (وصف المرض)
- dispensing_date (تاريخ الصرف)
```

#### 2. Medicines (الأدوية)
```
- id (معرف فريد)
- patient_id (معرف المريض)
- medicine_number (رقم الدواء)
- commercial_name (الاسم التجاري)
- unit (الوحدة)
- quantity (الكمية)
- expiry_date (تاريخ الانتهاء)
```

#### 3. Medicine_Receivers (مستقبلو الأدوية)
```
- id (معرف فريد)
- patient_id (معرف المريض)
- receiver_name (اسم المستقبل)
- relationship (صلة القرابة)
- card_type (نوع البطاقة)
- card_number (رقم البطاقة)
- phone (رقم التواصل)
```

#### 4. Documents (الوثائق)
```
- id (معرف فريد)
- patient_id (معرف المريض)
- id_card (بطاقة الهوية)
- prescription (روشتة الدواء)
- examination_form (ورقة المعاينة)
```

## 🔌 API Endpoints

### المرضى
- `GET /api/patients` - الحصول على جميع المرضى
- `GET /api/patients/:id` - الحصول على بيانات مريض محدد
- `POST /api/patients` - إضافة مريض جديد
- `PUT /api/patients/:id` - تحديث بيانات المريض
- `DELETE /api/patients/:id` - حذف المريض

### الإ��صائيات
- `GET /api/statistics` - الحصول على الإحصائيات العامة

### الاستيراد والتصدير
- `GET /api/export` - تصدير البيانات إلى Excel
- `POST /api/import` - استيراد البيانات من Excel

## 📝 ملاحظات مهمة

1. **البيانات تُحفظ محلياً** في ملف قاعدة البيانات `medicine_bank.db`
2. **لا يتطلب إنترنت** للعمل (Offline Application)
3. **واجهة عربية كاملة** من اليمين إلى اليسار (RTL)
4. **أسماء الأدوية بالإنجليزية** كما هو مطلوب
5. **النسخ الاحتياطية** يمكن أخذها بتصدير Excel
